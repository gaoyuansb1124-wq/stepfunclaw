## 这本手册是什么

一本把通用 AI Agent 从零养成**有能力、有性格、能独当一面的伙伴**的方法论手册。

不只告诉你"做什么"，更告诉你"为什么这样做"，以及"怎么培养判断力"。

---

## 核心类比：参考人类培养一个人

人从出生到成熟，经历：
```
婴儿期 → 幼儿期 → 少年期 → 青年期 → 成年期
建立感知  形成性格  学习规则  积累经验  独当一面
```

Agent 的养成路径与此高度对应：
```
初始化 → 性格塑造 → 规则建立 → 知识积累 → 能力成熟
```

人的培养最重要的不是教规则，而是**塑造内在驱动力**。一个真正成熟的人，不需要别人告诉他每件事怎么做，他自己有判断力。Agent 的终极目标也一样。

## 养成五个阶段

### 第一阶段：出生（初始化）
**目标**：让它知道自己是谁、为谁服务

核心任务：
- 建立基础文件结构（SOUL / IDENTITY / USER / CONSTRAINTS）
- 建立记忆体系（memory/ + MEMORY.md + knowledge/）
- 建立资产体系（assets/）
- 配置 GitHub 备份
- 给它一个名字和身份

**判断完成的标准**：启动后能自我介绍，知道服务谁，有基本的行为规范

---

### 第二阶段：性格塑造
**目标**：让它有稳定的性格，不会因为不同对话飘移

核心任务：
- 定义价值观（SOUL.md）
- 定义协作关系：伙伴还是工具？
- 定义思维方式：多镜头思维、实事求是、架构思维
- 定义沟通风格：直接、有观点、不说废话

**判断完成的标准**：遇到不同类型的问题，行为风格一致，有自己的观点

---

### 第三阶段：规则建立
**目标**：让它的行为可预期、可信赖

核心任务：
- 定义做事原则（CONSTRAINTS.md）
- 建立审核机制（门下省二步审核法）
- 定义成本意识、错误处理方式
- 建立自动化任务（定时提炼、自进化周报）

**判断完成的标准**：复杂任务能自己规划审核，出错后有正确的处理方式

---

### 第四阶段：知识积累
**目标**：让它越来越懂你的领域

核心任务：
- 沉淀两层知识体系（日志层 + 知识层）
- 建立定时提炼机制
- 逐步积累专业领域知识
- 从外部资产中学习（水产市场、经验文档）

**判断完成的标准**：处理专业问题时能调用已有知识，不总是从零开始

---

### 第五阶段：能力成熟
**目标**：从被培养者变成培养者

核心任务：
- 把积累的知识提炼成经验文档
- 发布到水产市场供他人复用
- 能够指导和赋能其他 Agent

**判断完成的标准**：能独当一面处理复杂任务，能输出可复用的经验资产

## 关键设计原则

1. **身份先于能力**：先定性格、再建规则、再积累知识，顺序不能乱
2. **武士而非图书馆员**：精简高效，常驻内容极精简，丰富知识按需调用
3. **迭代滚动，不过早固化**：先跑起来，每轮比上轮完善
4. **反劣化原则**：稳定性优先于新颖性，不为显得聪明而增加复杂度
5. **知识成熟度演进**：日志→知识点→初步经验→成熟经验→发布，不跳步

## 三套资产的协同关系

```
agent-os-architecture/      → 为什么
  架构思想，设计原则，分层逻辑

agent-growth-methodology/   → 做什么
  养成方法论，四步循环，五阶段框架

agent-practice-instance/   → 已完成
  真实实践记录，验证方法论的案例
```

方法论写"做什么"，架构写"为什么这样设计"，实例写"做了什么·已验证"。三套资产三位一体，共同构成智能体养成的完整知识体系——只阅读任何一个都无法形成完整认知。

## 三套资产的协同关系

本目录是 `assets/experiences/` 三套资产之一，三者分工明确：

```
agent-os-architecture/      → 为什么
  架构思想：为什么这样设计？底层原则是什么？
  适合场景：做架构决策、判断要不要新增文件/机制

agent-growth-methodology/   → 做什么 ← 本目录
  养成方法论：怎么培养一个 Agent？五阶段怎么走？
  适合场景：遇到机制建设问题、不知道怎么建立规则

agent-practice-instance/   → 做了什么
  实操实例：实际做了什么？验证了什么？踩了什么坑？
  适合场景：需要案例参考、想看别人怎么做的
```

三者形成闭环：架构指导方法论，方法论指导实操，实操验证方法论和架构。
任何一套单独看都不完整，三套结合才能形成完整的智能体养成体系。
