### 教训

**[ERR] 安装 skill 时 references 补写了但没告知用户内容是推测的**
- 正确做法：明确说"详细文档没拿到，以下内容是我补写的，可能不准确，请提供官方文档验证"

**[ERR] 有现成文档没有先读，凭印象乱猜字段名**
- `type` → 实际是 `note_type`
- `url` → 实际是 `link_url`
- 正确做法：动手调用 API 前，必须先找到并读懂官方文档

**[ERR] 出错后继续乱试而不是停下来找根因**
- 正确做法：第一次调用失败 → 停下来 → 找文档 → 理解后再试

### 正确的 API 调用流程
1. 先获取并读懂官方文档
2. 确认字段名、数据类型、必填项
3. 写好请求后一次调用
4. 失败了看错误码，对照文档排查

### 文档处理基本逻辑
收到文档先预处理转 Markdown，再做后续分析。
- docx → pandoc（最优）
- 网页/微信 → 浏览器转 md
- pdf文字版 → pymupdf
- 扫描版pdf/图片文字 → 告知用户有 token 成本
- pptx/xlsx → 直接 Python 提取，不转 md
详见 AGENTS.md「文档处理基本逻辑」章节

### 浏览器使用经验（详见 memory/browser-experience.md）
- 必须先 start 再 open，不能假设浏览器已运行
- web_fetch 用于静态页面，SPA/JS 渲染用后台浏览器
- 微信公众平台已在 openclaw profile 登录，cookie 持久保存
- 用 targetId 串联同一标签页的操作，tab not found 时重新 open

### Get笔记关键字段（已修正存档）
- 保存笔记：`note_type`（不是 type）
- 保存链接：`link_url`（不是 url）
