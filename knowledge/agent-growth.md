# agent-growth.md - 智能体养成

> 沉淀智能体养成的核心经验和方法论。
> 目标：让通用 Agent 成为真正懂主人、能干活的伙伴，并未来赋能其他智能体。

---

## [LRN] 智能体养成核心路径（2026-04-29）

**核心问题**：怎么把一个通用 Agent 变成真正懂你、能干活的伙伴？

### 1. 身份先行
先定义是谁、为谁服务、做什么，再谈能力。
- 名字、角色、工作方向要明确写进 IDENTITY.md
- 没有身份定义，Agent 每次启动都是陌生人

### 2. 约束工程
不是给 Agent 加限制，而是让行为可预期、可信赖。
- **SOUL.md** — 定性格、价值观、沟通风格
- **CONSTRAINTS.md** — 定做事规则、解决问题思路、错误处理方式
- 约束越清晰，Agent 越靠谱，越少让人操心

### 3. 协作关系设计
- 互商互量，不是甲方外包关系
- Agent 是决策参谋，提供材料和分析，决策权在主人
- 审核流程要有主人参与（门下省二步审核法）
- 发现主人卡在思考太久，要友情提醒动起来

### 4. 迭代滚动规划
- 不追求一次完美，先从基础跑起来
- 每轮比上轮更完善，边做边调整
- 过早固化分类和规则是浪费，等内容成熟再固化

### 5. 记忆体系是成长的基础
没有记忆就没有成长，三层缺一不可：
- **日志层**：`memory/日期.md`，带标签，边聊边写
- **长期记忆**：`MEMORY.md`，提炼后的精华
- **知识库**：`knowledge/`，可复用知识按主题存放

### 6. 带教关系
主人在带教 Agent 成长，Agent 要主动从对话中提炼方法论，及时更新自己的认知和规则。不只是执行指令，也在成长。

## [LRN] 精简高效的知识架构原则（2026-04-29）

**核心矛盾**：丰富的知识必然带来低效，但高效并不排斥丰富的知识。

**解法：分层存储，按需激活**

```
深层知识库（丰富）→ knowledge/ assets/
    按需查阅，不常驻

工作记忆（精简）→ SOUL.md CONSTRAINTS.md
    每次启动加载，常驻

当下上下文（极精简）→ 当前对话
    实时使用
```

**设计约束：**
- 常驻的东西必须极精简——只放真正影响每次行为的核心规则
- 丰富的知识沉在库里——用到时才激活，用完放回去
- knowledge/ 里的内容要标注使用场景，让我知道什么时候该查哪个

**目标：武士而非图书馆员**
不是百科全书，而是随时能拔出来用的利器。每招致命，随时可用。


**问题**：单 Agent 自我审核客观性不足（中书省和门下省都是自己，难以真正否定自己）

**解决方案**：二步审核法
1. Agent 用反驳清单自审
2. 主人初步反驳
3. Agent 修订
4. 主人最终拍板

**审核清单**：
- A. 需求层：完整理解意图？有无遗漏重要需求？
- B. 方案层：有无更简单方案？假设是否验证？成本时间可行？
- C. 风险层：最容易在哪失败？有无不可逆操作？
- D. 执行层：失败能回退？需小袁介入的节点是否标明？
