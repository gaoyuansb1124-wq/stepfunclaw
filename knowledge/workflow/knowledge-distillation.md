# 知识沉淀工作流

更新日期：2026-04-30

---

## 知识流转路径

```
原始材料（对话/文档/输入）
    ↓ 知识提取（打标签：[LRN][ERR][DEC][FEAT]）
    ↓
knowledge/（知识库）
— 存放迭代中的知识点，按主题分类，持续更新
    ↓ 知识点成熟（多次验证/实践印证）
    ↓
assets/experiences/（经验库）
— 成熟经验，可选择发布到水产市场
— 未成熟的继续在 knowledge/ 迭代
    ↓ 特别重要的经验
    ↓
assets/experiences/agent-growth-handbook/
— 养成手册，包含思路方法+案例
— 可复用，有商业化潜力，单独维护版本管理
```

---

## 触发时机

### 场景一：对话结束时（实时）
嵌入质检清单第6项：
```
□ 知识沉淀：本次对话有没有值得写进 knowledge/ 的知识点？
  → 有：立即写入对应主题文件
  → 没有：跳过
```

### 场景二：每日定时（异步）
- cron 每天22点自动触发
- 扫描当日 memory/YYYY-MM-DD.md
- 提炼带标签内容 → 写入 knowledge/

### 场景三：经验成熟时（按需）
- 判断标准：经过多次实践验证 + 有案例支撑 + 逻辑自洽可复用
- 触发方式：我主动提议，或小袁告知
- 动作：从 knowledge/ 提炼 → 写入 assets/experiences/

---

## 标签规范

| 标签 | 含义 |
|------|------|
| [LRN] | 学到的知识/经验 |
| [ERR] | 错误/踩坑 |
| [DEC] | 重要决策 |
| [FEAT] | 新功能/新能力 |

---

## 知识成熟度判断标准

- 经过多次实践验证（不是只出现过一次）
- 有具体案例支撑
- 逻辑自洽，可独立复用
- 满足以上三条 → 可提炼进 assets/experiences/
