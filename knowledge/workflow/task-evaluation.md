# 任务复杂度评估工作流

更新日期：2026-04-30

---

## 触发条件

S0 判断不是简单任务时，进入此流程。

---

## 执行流程

### Step1：五维打分

| 维度 | 1分 | 5分 |
|------|-----|-----|
| 步骤数 | 1步 | 5步以上 |
| 知识域 | 单一领域 | 多个领域 |
| 不确定性 | 全部已知 | 大量未知 |
| 失败代价 | 可轻松回退 | 不可逆 |
| 工具链 | 单工具 | 多系统协调 |

总分 = 五维之和（最低5分，最高25分）

### Step2：决策

| 总分 | 难度 | 决策 |
|------|------|------|
| 5~8分 | 低 | 直接执行，静默执行 |
| 9~15分 | 中 | 轻量规划，列出步骤边做边调整 |
| 16~25分 | 高 | 完整规划，中书省→门下省→六部执行 |

### Step3：向用户说明（一行）

格式：
> **任务评估：** 复杂度X/25分，[直接执行/轻量规划/完整规划]，[🟢低/🟡中/🔴高/⚫极高]消耗

示例：
> **任务评估：** 复杂度8/25分，直接执行，🟢低消耗

---

## >15分时：中书省规划格式

```
【规划】
任务摘要：{一句话概括}
子任务清单：
1. [部门] 任务描述 — 复杂度：低/中/高
2. [部门] 任务描述 — 复杂度：低/中/高
依赖关系：{先后顺序}
```

## 门下省审核清单

```
□ 完整性：所有需求覆盖？
□ 合理性：拆解粒度恰当？
□ 正确性：部门分配准确？
□ 可行性：方案可执行？
□ 无冗余：没有重叠任务？
```

全部通过 → 执行
不通过 → 重新规划（最多2次）
