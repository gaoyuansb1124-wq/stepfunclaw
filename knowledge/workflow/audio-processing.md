# 音频处理工作流

更新日期：2026-04-30

---

## 完整工作流

```
音频输入
    ↓ 判断来源

【方式二（默认）】                    【方式一（备用）】
小袁在Get笔记处理好                   小袁发音频路径给我
→ 告诉我笔记ID/链接                  → 我调API上传Get笔记
  或"去读最新一条"                   → Get笔记处理完
    ↓                                → 我读回结果反馈
我调API读取笔记内容
（转写原文+摘要，Get笔记已处理好）
    ↓
【外部音频文件（非Get笔记来源）】
faster-whisper 本地转写（免费）
    ↓
    ↓ 判断意图

整理会议纪要          提炼学习笔记          让我了解小袁
    ↓                     ↓                    ↓
输出会议纪要          提炼要点存Get笔记    我直接学习消化
回复/存Get笔记                           更新MEMORY.md/日记
                                        （不做二次加工，
                                         除非小袁特别要求）
```

---

## 三大场景

### 场景一：会议录音 → 会议纪要

```
Get笔记录音（或本地录音文件）
    ↓ 读取转写内容
    ↓ 按会议纪要格式整理
    输出：时间/参与人/讨论要点/决议/待办
    ↓
回复 或 存入Get笔记
```

### 场景二：学习录音 → 学习笔记

```
学习音频（录音/外部文件）
    ↓ 读取或转写文字
    ↓ 提炼知识要点/整理笔记
    ↓
存入Get笔记（结合笔记工作流）
```

### 场景三：Get笔记记录 → 我来学习

```
小袁分享Get笔记录音记录
    ↓ 我调API读取笔记内容
    （转写原文 + AI摘要，Get笔记已处理好）
    ↓ 我直接学习消化，不做二次加工
    ↓
更新 MEMORY.md / 日记
（除非小袁特别要求进一步加工）
```

---

## 工具分工

| 场景 | 工具 | 成本 |
|------|------|------|
| Get笔记已处理好 | Get笔记 API 读取 | 极低 |
| 外部音频文件转写 | faster-whisper 本地 | 免费 |
| 上传音频到Get笔记 | Get笔记上传API | 极低 |

---

## 方式选择原则

**默认方式二**（小袁在Get笔记处理，告诉我读哪条）：
- 流程最轻
- Get笔记已转写+摘要，我不重复处理
- token消耗极低

**方式一备用**（小袁发音频给我）：
- 小袁懒得开Get笔记时使用
- 我上传→Get笔记处理→我读回结果

| 读Get笔记+整理 | 🟢 低 | 只读文字，极低消耗 |
| 外部音频转写+处理 | 🟢 低 | faster-whisper本地免费 |
