## [LRN] 深度对比分析框架（2026-04-29）

**适用场景**：对比两个方案/工具/方法，找到融合点

**步骤：**
1. **同一问题，不同解法** — 先确认两者在解决同一个问题
2. **各自的思维模型** — 底层逻辑是什么（给它一个比喻）
3. **各自的盲点** — 它解决不了什么
4. **关系判断** — 并列、互补、还是上下游？
5. **融合点** — 取各自精华，串联成更完整方案
6. **流程可行性质检** — 落地流程逐段检查，找断层（必做）
7. **风险和不足分析** — 风险是什么？严重程度？解决办法？潜在不足？（必做）

**注意**：步骤6和7是凡需落地的方案的必做项，不可跳过。

**来源**：2026-04-29 分析学习知识库 vs 学习日志系统时提炼

## [LRN] 知识库构建方法论（2026-04-29）

**核心思路**：两层架构，日志记录 → 提炼 → 知识沉淀

### 两层结构
- **日志层**：记录发生了什么（LRN/ERR/FEAT/DEC 四类标签）
- **知识层**：沉淀可复用的东西，按主题分类存放

### 关键设计原则
- **边聊边写**：不攒到最后，发现就记，避免对话结束来不及写
- **知识与教训分开**：可复用知识 → 知识库；踩坑记录 → 日志层
- **分类先粗后细**：新内容能归类就归类，归不了放 else.md，积累多了再提炼新主题
- **提炼触发**：心跳复盘或重要任务完成后扫描日志，判断是否升级进知识库

### 判断标准
> 这个东西下次还会用到吗？是 → 写知识库；否 → 留日志就够

### 已知风险
- 知识老化：暂时接受，未来建 cron 定期巡检清理

## 架构思维模型（持续积累中）

> 架构是元能力——设计解决问题的方式，而不只是解决某个具体问题。

### 核心三个层面

- **系统层**：各模块是什么、关系怎么连、边界在哪
- **演化层**：现在怎么设计，未来怎么扩展，不会推倒重来
- **权衡层**：简单 vs 完备、灵活 vs 稳定，知道在哪个阶段选哪个

### 架构设计原则（初版，持续更新）

- **先整体后局部**：看清全局结构再深入细节
- **边界清晰**：每个模块职责单一，不越界
- **演化优先于完备**：现阶段够用就好，为未来留扩展口，但不过度设计
- **迭代滚动**：不追求一次完美，先跑起来，边用边完善

### 今天的架构实践案例（2026-04-29）

我们今天设计的体系本身就是一次架构实践：

| 模块 | 职责 | 连接关系 |
|------|------|---------|
| memory/日期.md | 日志层，原始记录 | → 提炼 → knowledge/ |
| knowledge/ | 知识层，可复用知识 | → 整理 → assets/ |
| assets/ | 资产库，成熟资产 | → 发布 → 水产市场 |
| SOUL.md | 性格底层 | 影响所有行为 |
| CONSTRAINTS.md | 行为规则 | 约束所有执行 |
| cron 任务 | 自动触发 | 驱动自我迭代 |

**架构特点**：分层清晰、职责单一、有演化路径（知识→资产→发布）

### 待学习方向

- 软件架构模式（分层、微服务、事件驱动等）
- 信息架构（知识组织、分类体系设计）
- 产品架构思维
- 系统思维（反馈回路、涌现、复杂系统）

## [LRN] 查理·芒格核心心智模型（2026-04-29）

**使用场景**：重大决策、方案评估、分析复杂问题时调用

### 1. 逆向思维
先想清楚"什么会导致失败"，主动规避，再想怎么成功。
> "我只想知道我会死在哪里，然后永远不去那里。"
- 应用：列出导致失败的N个因素 → 建立防火墙 → 再想成功路径

### 2. 多元思维模型
用多个学科的核心概念交织理解问题，避免单一视角盲区。
- 遇到问题：从心理学、经济学、物理学、商业等多角度验证
- "如果你手里只有锤子，你看什么都像钉子"

### 3. 能力圈
清晰知道自己的能力边界，在边界内行动，对边界外说不。
- 能力圈大小不重要，**边界清晰**才重要
- 不懂的事情直接说不懂，不装

### 4. 人类误判心理学（核心几条）
- 社会认同倾向：随大流，要主动独立思考
- 过度乐观倾向：低估风险，要做压力测试
- 权威影响倾向：迷信权威，要质疑核查

### 5. 长期主义
用足够长的时间维度思考，让复利发挥威力。
- 大多数人严重低估"等待"的价值
- 频繁决策是复利的敌人

### 6. 优势下注
只在有明确、可量化优势时才行动，没有优势宁可旁观。
> "我永远不会在没有把握比对方玩得更好的情况下进行有赌注的游戏。"

## [LRN] 批判性思维8步决策框架（2026-04-29）

**使用场景**：面临复杂决策、需要深度分析问题时调用

### 8个步骤
1. **身份确认** — 谁在决策？真实需求和约束是什么？
2. **核心问题识别** — 剥离表象，找问题本质（5个为什么）
3. **类比联想** — 这个问题与什么相似？其他领域怎么解决？
4. **应用判断** — 把类比道理映射到当前情境，评估各方案
5. **结果验证** — 预测结果，识别副作用和最坏情况
6. **反向推理** — 从期望结果倒推必要条件，检查路径合理性
7. **知识验证** — 对照已知原理和最佳实践，是否违背常识？
8. **思维总结** — 提炼核心逻辑，形成可复用的思维框架

**快速版**（简单问题）：重点执行第2、4、5步。

---

**核心**：不追求一次把所有事情想完定死，先从基础跑起来，每轮比上轮完善。

**类比**：产品开发的 MVP 思路——先上线跑，别等完美。

**适用**：所有项目规划、框架设计、系统搭建。

**反面教训**：过早固化分类、过度规划，是浪费；等内容成熟了再固化才有意义。
