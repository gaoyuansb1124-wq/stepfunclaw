# 学习知识库 · 总目录

> 适用版本：v3.0.0
> 这是瑶瑶自己的知识积累，像一个积水池，越积越多
> 存的是可复用的知识，不是踩坑记录（那些交给记忆系统）
> 永久保存，不受记忆压缩影响

---

## 知识库设计理念

**知识库 vs 记忆系统：**

| 维度 | 知识库（蓄水池） | 记忆系统（错题本） |
|------|----------------|------------------|
| 内容 | 可复用的知识、手法、经验 | 踩坑记录、错误教训 |
| 用途 | 执行任务前查阅 | 回顾历史时查阅 |
| 保存 | 永久，不压缩 | L2 压缩后精华入 L3 |

**一句话：知识库存知识，记忆系统存教训。**

---

## 收录标准

| 适合放进来 | 不适合放进来（去记忆系统） |
|-----------|--------------------------|
| 学到的专业手法/技巧 | 踩过的坑和错误 |
| 可复用的方法论 | 详细的过程日志 |
| 领域知识精华提炼 | 临时的想法或尝试 |
| 成功的经验模式 | 失败教训 |

**判断方法：是"知识"还是"教训"？**

---

## 读取规则

> ⚠️ **强制规则：执行任务前必须先查知识库！**
> 不读知识库 = 重复学习 = 效率低下

Agent **不需要每次都读整个知识库**，按需查对应文档：

| 任务场景 | 必须读取 | 不读的后果 |
|---------|---------|-----------|
| 写威胁情报报告 | threat-intelligence-knowledge.md | 表达平庸，没有积累 |
| 技能开发 | skill-knowledge.md | 重复摸索已知方法 |
| 理解记忆系统 | memory-governance.md | 混淆知识库和记忆系统 |
| 不确定要查哪篇 | 先读 INDEX.md | — |

**正确姿势：任务开始 → 查知识库 → 执行 → 学到新知识 → 写入知识库**

## 心跳锚定机制

每4小时心跳唤醒，检查：
1. 身份锚定（SOUL/IDENTITY/USER）
2. 知识库检查（有没有新知识要写进去）
3. 记忆系统检查（有没有踩坑要记录）
4. 风格检查（有没有跑偏）

详细说明见 SKILL.md 第四节。

## 知识库怎么长

- 每学到一样知识 → 往对应的文档里加
- 掌握新方法 → 往 skill-knowledge.md 补充
- 单篇超过3000字 → 拆分或创建新文档
- 知识库越大 → Agent 越"有学问"

### 何时创建新分类文档

**判断标准**：遇到新任务类型时问自己：
> "这类知识放在现有文档里合适吗？"

| 情况 | 动作 |
|------|------|
| 现有文档能涵盖 → 往现有文档加 |
| 新领域/新任务类型，现有文档都不贴切 → **创建新文档** |
| 一个文档超过 3000 字 → 考虑拆分 |

**创建新文档的步骤**：
1. 在 `~/.qclaw/skills/learning-knowledge-base/` 下创建 `新领域-knowledge.md`
2. 写上简介和收录标准
3. 更新 INDEX.md 的"当前文档"表
4. 记录到 MEMORY.md 技能清单

**可能的新分类示例**：
- `browser-automation-knowledge.md` — 浏览器自动化知识
- `docx-writing-knowledge.md` — Word 文档生成知识
- `api-integration-knowledge.md` — API 调用知识
- `prompt-engineering-knowledge.md` — 提示词优化知识

## 知识库 vs 记忆系统 快速判断

| 问题 | 答案 |
|------|------|
| 我学到了什么？ | → 知识库 |
| 我犯了什么错？ | → 记忆系统 |
| 这个方法以后还能用吗？ | → 知识库 |
| 这个错误以后要避免吗？ | → 记忆系统 |
| 这是领域知识吗？ | → 知识库 |
| 这是个人教训吗？ | → 记忆系统 |
