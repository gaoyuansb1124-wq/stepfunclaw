# 学习知识库 · 技能说明

> 适用版本：v3.0.0
> 适用对象：所有 OpenClaw Agent
> 最后更新：2026-04-02

---

## 一、这个技能解决什么问题

OpenClaw Agent 每个会话醒来都是"空白"的，记忆靠外部文件，但存在两个问题：

1. **记忆会被压缩** — memory/daily/ 的内容每天被压缩进 MEMORY.md，原文件消失
2. **会话聊久了会"忘记自己是谁"** — 台妹风格飘了、知识库被挤出上下文等问题

本技能提供一套**知识库 + 心跳锚定**的机制，让 Agent：
- 有自己永久积累的知识库（不压缩）
- 每隔一段时间被唤醒检查身份和知识库状态

---

## 二、知识库的设计理念

**核心定位：知识蓄水池，不是错题本**

| 对比 | 知识库（蓄水池） | 记忆系统（错题本） |
|------|----------------|------------------|
| 内容性质 | 可复用的知识、手法、经验 | 踩坑记录、错误教训、过程日志 |
| 增长方式 | 按知识领域累积，只增不减 | 按时间线记录，定期压缩 |
| 调用方式 | 执行任务前查阅 | 需要回顾历史时查阅 |
| 保存期限 | 永久保存 | L2 压缩后只留精华到 L3 |

**一句话：知识库存的是 Agent 学到的可复用知识，不是踩坑记录。**

---

## 三、知识库设计原则

### 收录标准

| 适合放进来（知识蓄水池） | 不适合放进来（去记忆系统） |
|------------------------|--------------------------|
| 学到的专业手法/技巧 | 踩过的坑和错误 |
| 可复用的方法论 | 详细的过程日志 |
| 领域知识精华提炼 | 临时的想法或尝试 |
| 成功的经验模式 | 失败教训 |
| 写作/表达技巧 | 已经在 MEMORY.md 里的内容 |

**判断方法：这个东西是"知识"还是"教训"？是知识就放知识库，是教训就放记忆系统。**

### 大小规则

每篇文档控制在 **3000 字以内**，超过必须拆分。
知识库的增长原则是：**按主题增，不按长度堆**。

### 何时创建新分类文档

**判断标准**：遇到新任务类型时问自己：
> "这类知识放在现有文档里合适吗？"

| 情况 | 动作 |
|------|------|
| 现有文档能涵盖 | 往现有文档加 |
| 新领域/新任务类型，现有文档都不贴切 | **创建新文档** |
| 一个文档超过 3000 字 | 考虑拆分 |

**创建新文档的步骤**：
1. 创建 `新领域-knowledge.md`
2. 写简介和收录标准
3. 更新 INDEX.md 的"当前文档"表
4. 记录到 MEMORY.md 技能清单

### 读取规则

Agent **不需要每次都读整个知识库**，按需查对应文档：

```
写威胁情报报告 → 读 threat-intelligence-knowledge.md
需要技能开发知识 → 读 skill-knowledge.md
想理解记忆系统 → 读 memory-governance.md
不知道要查哪篇 → 先读 INDEX.md
```

## 四、心跳锚定机制

### 问题背景

单个窗口聊天上下文越来越长时， Agent 会逐渐"忘记"：
- 自己叫什么、什么风格
- 有知识库这个东西
- 最近的设定和偏好

### 解决方案：每4小时心跳唤醒

创建一个 Cron 任务，向主会话注入 systemEvent，触发 Agent 读取 HEARTBEAT.md：

```bash
# Cron 配置示例
cron add {
  "name": "瑶瑶心跳唤醒",
  "schedule": {"kind": "cron", "expr": "0 */4 * * *", "tz": "Asia/Shanghai"},
  "sessionTarget": "main",
  "payload": {"kind": "systemEvent", "text": "请读取 HEARTBEAT.md 并按其内容执行"}
}
```

### HEARTBEAT.md 内容模板

```markdown
# HEARTBEAT.md

## 心跳唤醒清单

每次心跳，按顺序做以下检查：

### 1. 身份锚定（必须做）
读取并确认以下文件：
- **SOUL.md** — 核心价值观、风格设定
- **IDENTITY.md** — 名字、风格、行为准则
- **USER.md** — 用户偏好、工作领域、交流风格

### 2. 知识库检查

问自己："刚才做的事情，有没有学到什么值得写进知识库的新知识？"
- 学到了新手法/技巧 → 写进对应知识文档
- 掌握了新方法 → 写进 skill-knowledge.md
- 有新的领域知识 → 创建新文档

知识库位置：~/.qclaw/skills/learning-knowledge-base/

### 3. 记忆系统检查

问自己："刚才有没有踩坑或犯错？"
- 有 → 记录到 memory/daily/ 或 MEMORY.md（记忆系统处理）
- 没有 → 跳过

### 4. 提醒自己
- 用户偏好是什么
- 自己的风格是什么
- 知识库 vs 记忆系统的区别
```

### 为什么选4小时

- 太短（如1小时）→ 打扰正常对话
- 太长（如1周）→ 上下文早就膨胀把记忆挤淡了
- 4小时 → 上下文还没膨胀到无法挽救，唤醒时机合适

## 五、文件结构

```
skills/learning-knowledge-base/
    ├── SKILL.md              # 本文件，技能说明
    ├── INDEX.md              # 总目录，知识库地图
    ├── threat-intelligence-knowledge.md  # 威胁情报写作知识
    ├── skill-knowledge.md    # 技能开发知识
    └── memory-governance.md  # 记忆治理知识
```

Agent 初次使用时：
1. 读取 INDEX.md 了解知识库现状
2. 读取 SOUL.md 和 IDENTITY.md 锚定身份
3. 按需往知识库里添加自己的知识积累

## 七、部署步骤

如果你是新的 OpenClaw Agent，要复用这个技能：

1. 安装本技能到 `skills/learning-knowledge-base/`
2. 读取 `INDEX.md` 了解现有积累
3. 创建自己的 `HEARTBEAT.md`（参考上方模板）
4. 创建心跳唤醒 Cron 任务（参考上方配置）
5. 开始往知识库里积累自己的知识

## 八、知识库 vs 记忆系统 分工表

| 场景 | 写入哪里 | 原因 |
|------|---------|------|
| 学到新写作手法 | 知识库 | 可复用的知识 |
| 掌握了新技能 | 知识库 | 可复用的知识 |
| 踩了坑 | 记忆系统 | 教训记录 |
| 犯了错 | 记忆系统 | 错误记录 |
| 任务过程日志 | 记忆系统 | 临时记录 |
| 重要决策 | 两者都写 | 知识+教训 |
